# 🐚 Giao Diện Dòng Lệnh (Shell) & Tài Khoản Quản Trị (Superuser)
> Trong bài học này, bạn sẽ học về Shell là gì, cách truy cập Shell và tìm hiểu về tài khoản Superuser (người dùng tối cao) trong Linux.

---

### 1. Shell Là Gì & Cách Truy Cập?

* **Khái niệm:** Shell là giao diện người dùng mặc định của hệ thống Linux.

* **Cách thức hoạt động:** 
    * Khi bạn kết nối với hệ thống Linux qua mạng, Shell là chương trình tự động khởi chạy và đóng vai trò là giao diện chính để bạn tương tác với hệ thống.

    * Nếu bạn đăng nhập vào giao diện văn bản (Text-based mode), bạn sẽ làm việc trực tiếp với Shell.

    * Nếu bạn dùng giao diện đồ họa (GUI), bạn cần mở một ứng dụng mô phỏng dòng lệnh gọi là Terminal Emulator (hoặc gọi tắt là Terminal) để truy cập Shell.

* **Cách mở Terminal trên giao diện đồ họa::** 

    * Trên AlmaLinux / Ubuntu: Nhấp vào menu Activities (Góc trên bên trái màn hình) $\rightarrow$ gõ terminal $\rightarrow$ nhấp chọn ứng dụng Terminal (hoặc nhấn Enter).

---

### 2. Sức Mạnh Của Dòng Lệnh (CLI) So Với Giao Diện Đồ Họa (GUI)
> Mặc dù giao diện đồ họa cũng được coi là một dạng "Graphical Shell", nhưng khi người ta nói đến "Shell", họ thường ám chỉ Giao diện dòng lệnh (CLI).

* **💡 Ví dụ về sức mạnh của Shell:** Nếu bạn có 100 file tài liệu và cần đổi tên tất cả chúng:
   * Trên GUI: Bạn phải nhấp từng file một, đổi tên, rồi lặp lại thao tác đó 99 lần nữa.
   *  Trên Shell (CLI): Bạn chỉ cần gõ một dòng lệnh duy nhất để đổi tên toàn bộ 100 file cùng một lúc.

---

### Tại sao Web Dev / Quản trị viên BẮT BUỘC phải làm chủ CLI?

* **Luôn có sẵn:Luôn có sẵn:** Hầu hết các phiên bản Linux dành cho máy chủ (Server Edition) không cài giao diện đồ họa (GUI) để tiết kiệm tài nguyên.

* **Kết nối từ xa:** Khi bạn truy cập vào máy chủ Linux qua mạng, bạn sẽ dùng SSH (Secure Shell) — giao diện này cho bạn quyền làm việc trực tiếp trên dòng lệnh.

* **Linh hoạt:** Bạn có thể kết hợp cả hai. Khi cần lướt web trên máy tính cá nhân, bạn mở Firefox (GUI). Nhưng khi quản lý server, cấu hình phần mềm hay xử lý hàng loạt file, dùng CLI hiệu quả hơn rất nhiều.

### 3. Con Trỏ Dòng Lệnh (Shell Prompt) & Ký Tự Nhận Biết
> Khi Shell khởi chạy, nó sẽ hiển thị Shell Prompt (con trỏ dòng lệnh) để chờ bạn nhập lệnh.

> Một Shell Prompt thông thường sẽ cung cấp các thông tin hữu ích: jason@linuxsvr:\sim\$ 

* **jason:** Tên người dùng hiện tại.
* **@:** Ký tự phân cách.
* **linuxsvr:** Tên của máy chủ Linux (Hostname).
* **~:** Thư mục hiện tại (Dấu ngã đại diện cho thư mục home)
* **$ / #:** Ký tự phân biệt quyền hạn:
    * Dấu $: Bạn đang đăng nhập bằng người dùng thông thường (Normal User).
    * Dấu #: Bạn đang đăng nhập bằng Superuser (Tài khoản Quản trị tối cao / root).

---

### 4. Tài Khoản Quản Trị Tối Cao (Superuser / Root)
> Khái niệm: Superuser trên Linux có tên tài khoản là root.

* **⚠️ Lưu ý tránh nhầm lẫn:** Thư mục / (Root directory): Thư mục gốc tối cao của hệ thống tệp.

    * **Tài khoản root (Root account):** Tài khoản người dùng có quyền hạn tối cao. (Tùy vào ngữ cảnh câu nói để phân biệt hai khái niệm này).

    * **Thẩm quyền::** Tài khoản root tương tự như tài khoản Administrator trên Windows. Người dùng root có thể làm mọi thứ trên hệ thống Linux.

    * **Quy tắc sử dụng::** 
        * Các công việc hàng ngày nên được thực hiện bằng Tài khoản thường.
        * Quyền root chỉ nên dùng cho các tác vụ quản trị hệ thống: Cài đặt ứng dụng hệ thống, bật/tắt các dịch vụ (ví dụ: khởi chạy Web Server như Nginx/Apache). Các ứng dụng thông thường như trình duyệt web không cần quyền root.

---

### 5. Ký Tự Đặt Biệt ~ (Tilda) & Thư Mục home
> Trong con trỏ dòng lệnh hoặc đường dẫn, dấu ngã (~) là ký tự viết tắt đại diện cho Thư mục nhà (Home Directory):

| Cú pháp | Đường dẫn đầy đủ tương ứng | Ghi chú |
| :---: | :--- | :--- |
| **~** | /home/jason | Thư mục home của tài khoản hiện tại (ví dụ: jason). |
| **~pat** | /home/pat | Thư mục home của người dùng pat. |
| **~root** | /root | Thư mục home riêng của tài khoản quản trị root. |
| **~ftp** | /var/ftp | Thư mục home của tài khoản dịch vụ FTP. |

### 📝 Tóm Tắt Bài Học

* 1. Shell là giao diện người dùng mặc định của Linux để tiếp nhận và thực thi câu lệnh.
* 2. Mở Terminal trên Linux GUI hoặc kết nối qua SSH đều giúp bạn truy cập vào Shell.
* 3. Shell Prompt hiển thị thông tin về User, Server, Thư mục hiện tại và Cấp độ quyền hành ($ là user thường, # là root).
* 4. root là Superuser có toàn quyền trên hệ thống. Dùng user thường cho công việc hàng ngày và chỉ dùng root khi thực hiện các tác vụ quản trị.
* 5. Dấu ~ là ký tự tắt đại diện cho thư mục home.
